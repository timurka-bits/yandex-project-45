# 🕵️‍♂️ Распознавание дипфейков (Deepfake Detection) | Командный проект

В данном репозитории представлено решение задачи бинарной классификации изображений на настоящие и сгенерированные (дипфейки). Основная сложность задачи заключается в **сильном дисбалансе классов** (41 500 настоящих против 8 500 фейковых изображений). 

Целевая метрика качества — **F1-score**.

Для достижения максимального результата мы применили двухуровневую архитектуру (стекинг), объединив 7 различных базовых моделей (CNN и Vision Transformer), разработанных участниками команды.

---

## 🎯 Результаты

*   **OOF F1 Ансамбля:** 0.96225
*   **Test F1-score:** **0.97482**
*   Модель демонстрирует высокую способность выделять дипфейки без излишнего количества ложных срабатываний (False Positives).

## 👥 Вклад участников команды (Базовые модели)

Для обеспечения максимального разнообразия (diversity) ансамбля каждый участник разработал свои версии моделей с уникальными архитектурными решениями, функциями потерь и аугментациями. Использование разных `SEED` также позволило внести разнообразие за счёт инициализации весов и формирования батчей.

### 👨‍💻 Тимур
Сфокусировался на классических свёрточных архитектурах, продвинутых методах аугментации и применении трансформеров.
*   **Простая модель (CNN):** 5-блочная VGG-подобная архитектура с `BatchNorm`. Для борьбы с дисбалансом использована `CrossEntropyLoss` со взвешенными классами (1:3.5).
*   **Сложная модель (CNN + CutMix):** Внедрена аугментация `CutMix` (смешивание пар изображений и их меток) для мощной регуляризации. Добавлен `Label Smoothing` для калибровки уверенности сети и `Random Erasing`.
*   **Модель ViT (Vision Transformer):** Использована архитектура `vit-tiny` на базе патч-эмбеддингов. Применён `CosineAnnealingLR` для плавного затухания learning rate и инициализация со статистикой ImageNet.

### 👨‍💻 Амир
Реализовал архитектуры с фокусом на специфические функции потерь и извлечение нетривиальных признаков (частотный анализ).
*   **Простая модель (CNN + Focal Loss):** Классическая CNN, где проблема дисбаланса решается не весами, а применением `FocalLoss` ($\alpha=0.75, \gamma=2.0$), заставляющей модель фокусироваться на сложных примерах.
*   **Сложная модель (4 Канала + FFT):** Инновационное решение. К стандартным RGB каналам добавлен **четвертый частотный канал**, полученный с помощью быстрого преобразования Фурье (FFT). Архитектура адаптирована: использован `GroupNorm` вместо `BatchNorm` и обобщённый пулинг `GeM Pooling` (Generalized Mean Pooling) для лучшего выделения локальных паттернов фейков.

### 👩‍💻 Яна
Сделала упор на оптимизаторы, современные блоки внимания и методы семплирования данных.
*   **Простая модель (CNN + SGD):** Базовая свёрточная сеть, обученная с помощью `SGD` с импульсом Нестерова (Nesterov momentum) и расписанием `ReduceLROnPlateau`. Использованы сбалансированные веса классов.
*   **Сложная модель (SE-Blocks + Depthwise):** Продвинутая легковесная архитектура. Использованы `Depthwise Separable` свёртки, `Skip-connections` (остаточные связи) и `SE-блоки` (Squeeze-and-Excitation) для поканального внимания. Для борьбы с дисбалансом применён `WeightedRandomSampler`, выравнивающий частоту показа классов прямо на этапе загрузки батчей. Применены жесткие аугментации (ColorJitter, RandomRotation).

## 🏗️ Архитектура Мета-модели (Стекинг)

Мы использовали стекинг второго уровня, чтобы объединить вероятности, выданные 7 базовыми моделями.

1.  **Генерация мета-признаков:** Каждая из 7 моделей делает предсказания вероятностей (класс 1) на отложенной `holdout` выборке.
2.  **Ансамбль 2-го уровня:** На полученных 7 признаках обучаются две разные мета-модели:
    *   **Logistic Regression:** С сильной регуляризацией (`C=0.01`) и штрафами за дисбаланс (`class_weight={0: 1, 1: 5}`).
    *   **LightGBM:** Компактные деревья (`max_depth=2`, `num_leaves=4`) с сильной регуляризацией (`reg_alpha=0.5`, `reg_lambda=0.5`) и сбалансированными весами.
3.  **Блендинг (Линейное смешивание):** Финальное предсказание получается путём взвешенного сложения OOF-вероятностей двух мета-моделей. 
    *   Оптимальные веса подобраны с помощью grid search по метрике F1: **0.65 (LR) + 0.35 (LGBM)**.
    *   Оптимальный порог отсечения (threshold) для перевода вероятностей в бинарные классы: **0.5465**.

## 🔮 Направления для улучшения (Future Work)

---

### 🔄 1. Кросс-валидационное обучение (K-Fold)
На данный момент предсказания базовых моделей для мета-уровня получались исключительно с фиксированного `holdout`. Это ограничивает объем данных для обучения мета-модели.

*   **Решение:** Использование **StratifiedKFold** для обучения всех базовых моделей.
*   **Результат:** Это позволит получить честные **OOF-предсказания (Out-Of-Fold)** на всём объеме тренировочных данных, увеличит выборку для мета-моделей и сделает итоговый ансамбль значительно более устойчивым к переобучению.

---

### 🖼️ 2. Test-Time Augmentation (TTA)
При обычном инференсе модель видит только одну "идеальную" версию изображения, что может приводить к ошибкам на пограничных объектах.

*   **Решение:** Применение **TTA** — генерация нескольких аугментированных копий одной и той же тестовой картинки прямо в момент предсказания.
*   **Результат:** Модель делает прогнозы для оригинала, зеркального отражения (horizontal flip) и слегка повернутых версий, после чего результаты усредняются. Это сглаживает случайные аномалии и повышает точность классификации на стыке классов.
