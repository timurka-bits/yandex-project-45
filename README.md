# Распознавание дипфейков (Deepfake Detection) Командный проект

В данном репозитории представлено решение задачи бинарной классификации изображений на настоящие и сгенерированные (дипфейки). Основная сложность задачи заключается в сильном дисбалансе классов (41 500 настоящих против 8 500 фейковых изображений). 

**Целевая метрика качества:** F1-score.

Для достижения максимального результата применена двухуровневая архитектура, объединяющая 7 различных базовых моделей, разработанных участниками команды.

---

## Результаты

* **OOF F1 Ансамбля:** 0.96225
* **Test F1-score:** 0.97482
* Модель демонстрирует высокую способность выделять дипфейки без излишнего количества ложных срабатываний.

## Вклад участников команды (Базовые модели)

Для обеспечения максимального разнообразия ансамбля каждый участник разработал свои версии моделей с уникальными архитектурными решениями, функциями потерь и аугментациями. Использование разных значений SEED позволило внести разнообразие за счёт инициализации весов и формирования батчей.

### Тимур
Фокус на классических свёрточных архитектурах, продвинутых методах аугментации и применении трансформеров.
* **Простая модель (CNN):** VGG-подобная архитектура с 5 блоками и с BatchNorm. Для борьбы с дисбалансом использован CrossEntropyLoss со взвешенными классами (1:3.5).
* **Сложная модель (CNN + CutMix):** Внедрена аугментация CutMix для мощной регуляризации. Добавлен Label Smoothing для калибровки уверенности сети и Random Erasing.
* **Модель ViT (Vision Transformer):** Использована архитектура vit-tiny на базе патч-эмбеддингов. Применён CosineAnnealingLR для плавного затухания learning rate и инициализация со статистикой ImageNet.

### Амир
Реализация архитектур с фокусом на специфические функции потерь и извлечение нетривиальных признаков.
* **Простая модель (CNN + Focal Loss):** Классическая CNN, где проблема дисбаланса решается применением FocalLoss, заставляющей модель фокусироваться на сложных примерах.
* **Сложная модель (4 Канала + FFT):** К стандартным RGB каналам добавлен четвертый частотный канал, полученный с помощью быстрого преобразования Фурье. В архитектуре использован GroupNorm вместо BatchNorm и GeM Pooling для лучшего выделения локальных паттернов фейков.

### Яна
Упор на оптимизаторы, современные блоки внимания и методы семплирования данных.
* **Простая модель (CNN + SGD):** Базовая свёрточная сеть, обученная с помощью `SGD` с импульсом Нестерова (Nesterov momentum) и расписанием `ReduceLROnPlateau`. Использованы сбалансированные веса классов.
* **Сложная модель (SE-Blocks + Depthwise):** Продвинутая легковесная архитектура. Использованы Depthwise Separable свёртки, Skip-connections и SE-блоки для поканального внимания. Для борьбы с дисбалансом применён WeightedRandomSampler, выравнивающий частоту показа классов на этапе загрузки батчей. Применены аугментации ColorJitter и RandomRotation.

## Архитектура Мета-модели

Для объединения вероятностей, выданных 7 базовыми моделями, использовался стекинг второго уровня.

1. **Генерация мета-признаков:** Каждая из 7 моделей делает предсказания вероятностей на отложенной holdout выборке.
2. **Ансамбль 2-го уровня:** На полученных 7 признаках обучаются две разные мета-модели:
   * **Logistic Regression:** С сильной регуляризацией и штрафами за дисбаланс.
   * **LightGBM:** Компактные деревья с сильной регуляризацией и сбалансированными весами.
3. **Блендинг (Линейное смешивание):** Финальное предсказание получается путём взвешенного сложения OOF-вероятностей двух мета-моделей. 
   * Оптимальные веса подобраны с помощью grid search по метрике F1: **0.65 (LR) + 0.35 (LGBM)**.
   * Оптимальный порог отсечения для перевода вероятностей в бинарные классы: **0.5465**.

## Как запустить финальный ансамбль

Для инференса требуются сохраненные веса мета-моделей и заранее сгенерированные .npy файлы предсказаний базовых сетей.

```python
import pickle
import numpy as np
import pandas as pd

# Загрузка мета-моделей
with open('logistic_regression_meta.pkl', 'rb') as f:
    lr_model = pickle.load(f)
with open('lightgbm_meta.pkl', 'rb') as f:
    lgb_model = pickle.load(f)

# Параметры блендинга
best_w = 0.65
best_th = 0.5465

# Матрица мета-признаков (test_probs_...npy из шага базовых моделей)
# X_test_meta = np.column_stack([...])
# X_test_meta_df = pd.DataFrame(X_test_meta, columns=feature_names)

test_probs_lr = lr_model.predict_proba(X_test_meta)[:, 1]
test_probs_lgb = lgb_model.predict_proba(X_test_meta_df)[:, 1]

final_probs = best_w * test_probs_lr + (1 - best_w) * test_probs_lgb
final_preds = (final_probs > best_th).astype(int)
```

## Результаты

---

### 1. Кросс-валидационное обучение 
На данный момент предсказания базовых моделей для мета-уровня получались исключительно с фиксированного holdout. Это ограничивает объем данных для обучения мета-модели.

*   **Решение:** Использование StratifiedKFold для обучения всех базовых моделей.
*   **Результат:** Это позволит получить честные OOF-предсказания на всём объеме тренировочных данных, увеличит выборку для мета-моделей и сделает итоговый ансамбль значительно более устойчивым к переобучению.

---

### 2. Test-Time Augmentation 
При обычном инференсе модель видит только одну идеальную версию изображения, что может приводить к ошибкам на пограничных объектах.

*   **Решение:** Применение TTA — генерация нескольких аугментированных копий одной и той же тестовой картинки прямо в момент предсказания.
*   **Результат:** Модель делает прогнозы для оригинала, зеркального отражения и слегка повернутых версий, после чего результаты усредняются. Это сглаживает случайные аномалии и повышает точность классификации на стыке классов.
